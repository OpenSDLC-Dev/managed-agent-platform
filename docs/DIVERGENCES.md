# DIVERGENCES.md — wire divergences & unconfirmed inferences

The single registry of (a) **deliberate divergences** from the Anthropic Claude
Managed Agents reference and (b) **inferences** about reference behavior made where
neither the SDK types nor the `ant` CLI can reveal it — each to be confirmed against a
real managed-agents recording. The verifier's wire-compat rung resolves intentional
mismatches against this file; an intentional mismatch missing here is a defect. New
divergences land here in the same PR that introduces them.

Format: **surface** — behavior. *Evidence.* Three sections: architecture/compatibility notes (not mismatches), CONFIRMED deliberate divergences, and INFERRED assumptions (each cross-linked to the issue tracking its confirmation).

## Architecture & compatibility notes (not divergences)

Facts that read like divergences but are not mismatches with the managed-agents wire —
recorded here so the allowlist stays honest.

- **Session resource (all session endpoints) — end-user binding** — Sessions carry no user_id field at all. Scoping keys are org/workspace/project (reserved, single-tenant defaults in v1); end-user-to-session ownership is an application-layer concern via session metadata and the audit-only created_by. This is a deliberate architectural divergence (explicitly from adk-go's AppName+UserID pattern; the platform stays user-agnostic). *Evidence: CLAUDE.md 'Non-negotiable design principles' #5; docs/HISTORY.md § "`internal/store` — Postgres schema + migrations" (sessions schema: 'audit-only created_by, no user_id'); CLAUDE.md standing product decisions*
- **GET /v1/sessions/{id}/events/stream — SSE preview delta frames** — Streaming preview deltas use type 'content_delta', never the Messages API's 'content_block_delta'. event_start/event_delta frames are stream-only, never persisted, and reconcile under the buffered event's pre-allocated id. *Evidence: CLAUDE.md 'Wire-compatibility rules'; docs/HISTORY.md § "`internal/events` + events API — append-only log, send/list, SSE stream (slice 3)" (events/preview.go: 'delta type is literally content_delta, NOT the Messages API's content_block_delta')*

## Deliberate divergences (CONFIRMED)

- **Session ID prefix parsing (all endpoints accepting a session id)** — Canonical prefix is sesn_; the alternate session_ form is accepted on input, mirroring the reference's dual form. *Evidence: CLAUDE.md 'Wire-compatibility rules'; docs/HISTORY.md § "`internal/domain` — Anthropic-native core types" (domain/id.go 'accepts the alternate session_ form on input'); docs/HISTORY.md § "`internal/api` + `cmd/controlplane` — wire-compatible control-plane CRUD"*
- **All endpoints — anthropic-version / anthropic-beta headers and ?beta=true** — anthropic-version and anthropic-beta request headers are accepted and ignored; ?beta=true is accepted where the reference honors it. No version negotiation is performed. *Evidence: CLAUDE.md 'Wire-compatibility rules'; docs/HISTORY.md § "`internal/api` + `cmd/controlplane` — wire-compatible control-plane CRUD" (api/server.go: '?beta=true and anthropic-* headers accepted and ignored')*

Additive platform surface (deliberate divergence goal: OTel built in). *Evidence: CLAUDE.md principle #3; docs/HISTORY.md § "`internal/events` + events API — append-only log, send/list, SSE stream (slice 3)" (events/span.go), docs/HISTORY.md § "BYOC worker — `traceparent` propagation across the process boundary (slice 8, PR C2b-2)" (PR C2b-2, one OTel trace across the process boundary)*

The reference runs tools on the host and canonicalizes/rejects escapes; here the container is the boundary, so a lexical check would be theatre. *Evidence: docs/HISTORY.md § "`internal/toolset` — the built-in tools (slice 6, third part)" (slice-6 toolset decisions, divergences from the SDK's tools/agenttoolset)*
- **grep built-in tool — implementation strategy** — One implementation over the image's GNU grep (PCRE where available, ERE fallback), not the reference's ripgrep-or-Go-walker pair with two behaviors. Requires GNU coreutils/grep/findutils in the sandbox image; a busybox-only image gets a clear tool error rather than degraded behavior. *Evidence: docs/HISTORY.md § "`internal/toolset` — the built-in tools (slice 6, third part)"*
- **web_fetch / web_search built-in tools** — Deferred though present in the wire's tool-config enum: enabling one offers the model nothing, and calling it returns an error result rather than hanging (they need the egress policy that does not exist yet). *Evidence: docs/HISTORY.md § "`internal/toolset` — the built-in tools (slice 6, third part)"* *Tracked: #47.*
- **bash tool result — stdout/stderr interleaving** — stdout and stderr are captured separately and the result carries stderr after stdout whole, not interleaved. The reference's bash runs on a PTY and merges them; matching that would require giving the sandbox a PTY. *Evidence: docs/HISTORY.md § "`internal/toolset` — the built-in tools (slice 6, third part)"*
- **bash tool — cross-call shell state persistence** — Statefulness (cd, exports, functions, options; restart:true wipes) is provided via a per-call snapshot/restore on the container's writable layer, not a resident shell. Documented divergences from a resident shell, each pinned by a test: the jobs table does not carry, plain (non-exported) variables do not carry, traps do not carry, xtrace does not carry, a timed-out call's mutations are dropped, a call whose snapshot never completes keeps the previous call's state, and the timeout bounds the whole call (restore+command+snapshot). *Evidence: docs/HISTORY.md § "`internal/sandbox/shell` — the persistent bash tool (slice 6, second part)" (internal/sandbox/shell; 'Divergences from a resident shell — all of them, each pinned by a test')*
- **write / edit built-in tools — atomicity** — Not atomic: the reference writes to a temp file and renames; sandbox.WriteFile extracts a tar onto the target, so a mid-transfer failure can leave a truncated file (contained to the failed write's own file since the executor serializes a session's tool calls). Atomicity deferred to the sandbox backend. *Evidence: docs/HISTORY.md § "`internal/toolset` — the built-in tools (slice 6, third part)" (toolset known limits)* *Tracked: #71.*
- **Permission gating — mixed always_ask / always_allow turns** — A turn calling both an always_ask and an always_allow tool gates the whole turn: even the allow tools wait for confirmation (no partial execution while a confirmation is outstanding). requires_action.event_ids still lists only the ask tools. Deliberate simplicity/safety decision; reference behavior for mixed turns is unspecified. *Evidence: docs/HISTORY.md § "permission policies + the confirmation round-trip (slice 7)" ('Design decision — a mixed turn gates everything, not just its ask tools')*
- **Environment worker keys — issuance** — Key issuance is a platform-owned primitive (EnsureEnvironmentKey: hash-only, revoke-others-on-re-mint, key value bound to one environment for life). The reference mints environment keys off-wire in its console — no SDK/CLI wire endpoint creates one — so a self-hostable platform must own this. The consuming side (Bearer -> environment) stays wire-locked by the real ant beta:worker. Currently there is also no operator surface invoking it (the slice-8 acceptance seeded a key directly into the DB); issuance UX is the tracked next task. *Evidence: docs/HISTORY.md § "work API — environment-key auth + `/work/poll` (slice 8, first part)" (slice-8 part-1 decisions); docs/HISTORY.md § "Slice-8 acceptance — real `ant beta:worker` end to end (run 2026-07-16) — ✅ passed", issue #43; anthropic-cli: worker only consumes --environment-key*
- **GET /v1/environments/{id}/work/poll — block_ms** — block_ms is accepted but the poll is non-blocking (returns immediately); the reference client already spaces empty polls with client-side jitter sleep, so the protocol is unchanged, only chattier. True long-poll on a work_items NOTIFY is deferred. *Evidence: docs/HISTORY.md § "work API — environment-key auth + `/work/poll` (slice 8, first part)"; docs/HISTORY.md § "BYOC worker — the lease loop + `cmd/worker` binary (slice 8, PR C2b)" (our worker paces empty polls with EmptyPollSleep)* *Tracked: #74.*
- **POST …/work/{work_id}/stop — response shape** — **Not a divergence** (entry retained because it reverses a previously CONFIRMED one). Success is a bodiless 204 with no Content-Type, matching the reference service. The earlier entry claimed 200 + the updated BetaSelfHostedWork on the grounds that the generated Work.Stop is typed *BetaSelfHostedWork and a 204 makes the SDK's decoder error — that measurement was real but it measured the *client*, not the service. The pinned SDK's own work poller says so outright: "Today the server returns 204 with no body / no Content-Type, and the strict Go decoder errors … for what is actually a successful call", a Go-only bug (TS/Python decode 204 fine) worked around with WithResponseBodyInto, and its TODO asks for the *spec* to declare 204 for a body "that the server never sends". Our BYOC worker adopts the same bypass. A conflicting stop (already past the requested transition) is still 409 invalid_request_error, which the reference worker ignores. *Evidence: anthropic-sdk-go v1.56.0 lib/environments/poller.go:439-465 (the comment and the WithResponseBodyInto workaround), lib/environments/worker_test.go:118-120 (HandleStop writes StatusNoContent); internal/worker/lease_test.go TestWorkerForceStopAcceptsNoContent reproduces the decoder error verbatim when the bypass is removed; CHANGELOG.md § [Unreleased] → Fixed* *Resolved: #27.*
- **Work-item stop — identity-blind hung-worker race** — Wire-settled: the work item carries no generation/version field and stop's body is {force} only (no ownership proof), so a lease-guarded stop is impossible. Residual documented race: a hung-then-revived worker could complete its tools before its heartbeat 412s and force-stop the replacement's reclaimed item, re-stranding the session — a limitation the reference shares. Full fix (a fresh work identity per hand-out) is a later hardening; a truly dead worker is fully covered by reclaim. *Evidence: docs/HISTORY.md § "work API — the work-item lifecycle: get / ack / heartbeat / stop (slice 8, second part)" (PR C3 reclaim paragraph, 'Wire-settled against the reference … Residual (documented, not a v1 blocker)'); anthropic-sdk-go betaenvironmentwork.go (no generation field)* *Tracked: #62.*
- **traceparent to the BYOC worker — storage and transport** — The turn's trace context is stored in a dedicated trace_context jsonb column on work_items (migration 0005) — diverging from the plan's sketch of riding work-item metadata, because metadata is now client-updatable (PR C-meta) — and emitted as traceparent/tracestate response headers on /work/poll, never in the wire work body. The reference worker ignores unknown headers; ours reads them. Additive, wire-compatible surface. *Evidence: docs/HISTORY.md § "work API — environment-key auth + `/work/poll` (slice 8, first part)", docs/HISTORY.md § "BYOC worker — `traceparent` propagation across the process boundary (slice 8, PR C2b-2)" (PR C2b-2, 'Why a column, not metadata')*
- **GET /v1/environments/{id}/work/stats — provenance** — BetaSelfHostedWorkQueueStats is a derived view over Postgres (depth/pending/oldest_queued_at from the work_items snapshot; workers_polling from a poll-time worker-id record, migration 0006, 30-second window), not Redis Streams consumer-group figures. The SDK's field docs are Redis-native (depth = lag, pending = PEL, workers_polling = consumers), all but confirming the reference queue is Redis Streams; same wire numbers, Postgres provenance. Interpretation baked in: the wire's 'acknowledged' is our Ack (queued->starting), so depth = queued items available (no reservation or lapsed lease) and pending = polled-but-not-acked. *Evidence: docs/HISTORY.md § "work API — the queue-stats endpoint (slice 8, PR C-stats)" ('Divergence noted: same wire numbers, Postgres provenance'); anthropic-sdk-go field docs cited there*
- **Helm chart (deploy/helm/managed-agent-platform) — plan-sketch divergences** — Two deliberate divergences from the plan's chart sketch: Postgres ships inline (StatefulSet, toggleable to externalDatabase.url) rather than as a subchart (an external chart-repo dependency undercuts air-gap self-hosting), and the optional gVisor RuntimeClass is deferred until the K8s provider sets runtimeClassName on sandbox pods (shipping it now would be unwired). The BYOC worker is intentionally out of the chart. *Evidence: docs/HISTORY.md § "Kubernetes sandbox provider (slice 9)"*
- **K8s sandbox backend — deadline and limited-networking fidelity vs Docker** — Two documented necessity divergences from the Docker backend's guarantees: (1) the exec-deadline verdict reads state on the sandbox's own filesystem (Kubernetes has no out-of-band exec handle), so a malicious command can tamper with it in both directions — overwriting the pid file after killing its watchdog to hide an overrun, or writing the watchdog's kill mark and contriving a SIGKILL to claim a timeout it never hit (the mark is what classifies a punctual kill here, since the liveness probe is itself an exec and answers a round trip late — #95, #110). Out of scope for single-tenant v1: honest runaways forge nothing, every term of the verdict only ever *adds* a timeout so a forged mark cannot withdraw one, and a forged timeout mislabels only the tenant's own tool call. Docker keeps no such in-pod state — its probe is a daemon call the sandbox cannot reach — and deliberately still does not; (2) the limited-networking route flush is not equivalent to NetworkMode none on every cluster (raw AF_PACKET sockets can reach the segment; only the main IPv4 table is inspected, so policy-routing CNIs and IPv6 need the reserved egress proxy). *Evidence: docs/HISTORY.md § "Kubernetes sandbox provider (slice 9)"*
